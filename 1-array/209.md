# Questions

Given an array of positive integers nums and a positive integer target, return the minimal length of a contiguous subarray [numsl, numsl+1, ..., numsr-1, numsr] of which the sum is greater than or equal to target. If there is no such subarray, return 0 instead.

Example 1:

Input: target = 7, nums = [2,3,1,2,4,3]
Output: 2
Explanation: The subarray [4,3] has the minimal length under the problem constraint.
Example 2:

Input: target = 4, nums = [1,4,4]
Output: 1
Example 3:

Input: target = 11, nums = [1,1,1,1,1,1,1,1]
Output: 0
 

Constraints:

1 <= target <= 109
1 <= nums.length <= 105
1 <= nums[i] <= 105
 

Follow up: If you have figured out the O(n) solution, try coding another solution of which the time complexity is O(n log(n)).

# Solution1-Violent_1

"If there is a problem, fix it"

So it is particularly redundant.
```
class Solution {
public:
    int minSubArrayLen(int target, vector<int>& nums) {
        int sum = 0;
        int num = nums.size();
        int len = num;
        bool flag = false;
        for(int i = 0; i<len;i++){
            for(int j = i; j<len;j++){
                sum += nums[j];
                if(sum>=target){
                    if((j-i+1)<=num) {
                        num = j-i+1;
                        flag = true;
                        break;
                    }
                }
            }
            if((!flag)&&(sum<target)) return 0;
            sum = 0;
        }
        return num;
    }
};
```

In this code, "if((j-i+1)<=num)" is used for judging if the variant "num", the length needed to output, is replaceable, and here is a bool variant just because, if not, the program will return 0 directly in some cases, which is wrong.

# Solution2-Violent_2
```
class Solution {
public:
    int minSubArrayLen(int target, vector<int>& nums) {
        int result = INT32_MAX;
        int sub = 0;
        int sum = 0;
        for(int i = 0; i < nums.size(); i++){
            for(int j = i; j < nums.size(); j++){
                sum += nums[j];
                if(sum >= target){
                    sub = j-i+1;
                    result = result < sub ? result : sub;
                    break;
                }
            }
            sum = 0;
        }
        return result==INT32_MAX ? 0 : result;
    }
};
```

# Solution3-Sliding Window
```
class Solution {
public:
    int minSubArrayLen(int target, vector<int>& nums) {
        int result = INT32_MAX;
        int sub = 0;
        int sum = 0;
        int head = 0;
        for(int i = 0;i<nums.size();i++){
            sum+=nums[i];
            while(sum>=target){
                sub = i - head + 1;
                result = result<sub? result : sub;
                sum-=nums[head++];
            }
        }
        return result==INT32_MAX ? 0 : result;
    }
};
```
## Note2:
The key of this way is about Sliding Window's head, when found a sum is bigger than target, maybe its no need to sum that much numbers, so here is a "head++", and remove the value of this index in the array.
